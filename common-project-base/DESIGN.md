# DESIGN.md

This file defines the visual identity and UI behavior contract for coding agents.

## Required agent behavior

For UI changes, agents must read this file before planning UI work, reuse the project component library first, implement loading/empty/error/success states where relevant, verify responsive behavior, and keep design tokens consistent with Tailwind/shadcn CSS variables.

## Tokens

Define project colors, typography, radius, spacing, elevation, and motion rules here.

## Component rules

Define how buttons, forms, tables, cards, navigation, dialogs, empty states, and feedback messages should look and behave.

## Anti-patterns

List visual/design patterns agents must avoid.
